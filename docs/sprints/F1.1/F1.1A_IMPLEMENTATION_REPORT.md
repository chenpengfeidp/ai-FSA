## Implementation Report

### Delivered

1. **Provider Registry** in `@fas/evidence` (`EvidenceProviderRegistry` + default registry)
   - Categories: Football / Market / Sentiment / Prediction / Internal
   - Connected: `football:api-sports`, `internal:recorded`
   - Extension stubs only: `market:the-odds-api`, `sentiment:stub`, `prediction:stub`

2. **Provider Capability** declarations + query
   - Kinds: player, coach, referee, venue, lineup, injury, recent_form
   - API-Sports declares support for all kinds; **ingestImplemented** only for `recent_form` (existing F.1 TEAM_FORM)
   - HTTP: `GET /api/providers`, `/connected`, `/capabilities?kind=`, `/:providerId`, `/:providerId/capabilities`

3. **Evidence Metadata** on every Evidence item
   - `providerId`, `source`, `freshness`, `confidence` (source confidence), `timestamp`, enriched `provenance` (`providerId`, `category`)
   - Defaults preserve backward compatibility via `resolveProviderFromSource`

4. **Import Pipeline**
   - Normalizer records provider metadata on create
   - Football match shell carries `providerSource` / method into MATCH_INFO
   - **No** Feature / Rule / Projection changes

5. **Workspace**
   - Evidence timeline + Evidence card show provider / source / method / confidence

### Explicitly not delivered (out of F1.1A)

- Player import
- Injury import
- Lineup import
- Rule changes
- Projection changes
- New packages / Engines

## Backward Compatibility

| Concern | Behavior |
|---|---|
| Existing `createEvidence` callers | Optional `providerId` / `confidence` / `timestamp`; defaults from `source` |
| Legacy `provenance` without category | Filled by `resolveProviderFromSource` |
| Postgres `value_json` envelopes | Old rows without new fields still parse; defaults applied on read |
| Feature extractor | Ignores new Evidence fields (payload-only math) |
| Analyze path | Unchanged requirements (MATCH_INFO + TEAM_FORM + STATISTICS) |

Source → provider defaults:

| `source` | `providerId` | category |
|---|---|---|
| `api-football` | `football:api-sports` | football |
| `fixture` | `internal:recorded` | internal |
| `the-odds-api` | `market:the-odds-api` | market |

## Migration Notes

1. No Prisma schema migration required — metadata lives in Evidence domain + `value_json` envelope.
2. Re-import recorded football matches to refresh MATCH_INFO provider fields (`api-football` on football bundles).
3. Clients consuming Evidence JSON should tolerate new fields; treat missing `providerId` as pre-F1.1A (API always emits after deploy).
4. Provider path params containing `:` should be URL-encoded (e.g. `football%3Aapi-sports`).
