## A. Automated smoke

```bash
pnpm --filter @fas/evidence build
pnpm --filter @fas/evidence-normalizer build
pnpm --filter @fas/provider-football test
pnpm --filter @fas/evidence-normalizer test -- availability-evidence
pnpm --filter @fas/web test -- explainable-report
```

Expect:

- Cassette `football:100001` normalizes to INJURY + SUSPENSION.
- Explainable report `availability.available === true` with summary counts.

---

## B. Provider → Evidence

1. Use recorded football provider (no live key required).
2. Import / normalize `football:100001`.

**Pass**

- Evidence includes `type: "INJURY"` (e.g. FC Seoul Forward / Hamstring).
- Evidence includes `type: "SUSPENSION"` (e.g. Ulsan Defender).
- `providerId` resolves to `football:api-sports`.

**Honest absence**

- Match without `availabilityAbsences` → zero Injury/Suspension Evidence.
- Do not invent Unknown or “all available”.

---

## C. Workspace → Report

1. Open Workspace for the imported match.
2. Confirm **出场可用性 / Availability**:

| Check | Expectation |
|---|---|
| Summary | Injury count · Suspension count |
| Injury list | Player, team, reason, provider source |
| Suspension list | Distinct from injury |
| Timeline | Titles **Injury** / **Suspension** |
| Absence path | “暂无可用性证据” + non-confirmation copy |
