## What is in scope

| Included | Excluded |
|---|---|
| Player Id, Name, Team, Position | Player Statistics |
| Number / Nationality / Photo (when present) | Rating, Injury, Suspension |
| Evidence type `PLAYER` | Expected Lineup |
| Workspace + Report presentation | Feature / Rule / Projection changes |

## Path under test

```text
Provider (recorded cassette or live /players/squads)
  → Normalizer (parsePlayers)
  → Evidence (type PLAYER)
  → Workspace (Players section)
  → Report (explainable report + timeline)
```

## B. Recorded Provider → Evidence

1. Ensure Football provider uses recorded cassettes (default private demo path; no live key required for this check).
2. Resolve match id `football:100001` (K League cassette).
3. Import / normalize the match into Evidence.

**Pass criteria**

- Evidence list contains one or more items with `type: "PLAYER"`.
- Payload includes at least `playerId`, `name`, `teamId`, `teamName`, `teamSide`.
- Optional fields (`position`, `number`, `nationality`, `photo`) appear only when present in the cassette.
- Provenance / provider resolution shows football provider (e.g. `football:api-sports` / `api-football`).

**Honest absence**

- A match shape without `players` must still import; zero `PLAYER` evidence (do not invent a squad).

## C. Workspace → Report UI

1. Start private stack as usual (`pnpm dev:api` / `pnpm dev:web` when validating interactively).
2. Open Workspace for an imported match that has PLAYER evidence (e.g. `football:100001` after import/analyze).
3. Confirm:

| Surface | Expectation |
|---|---|
| Players section | Home/away lists with name; optional number, position, nationality, photo |
| Evidence timeline | Entries titled **Player**, type tag `PLAYER`, provider source line |
| Venue / prediction / features / rules | Unchanged behavior from prior slices |
