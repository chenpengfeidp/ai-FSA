## Implementation Report

### Goal

End-to-end **Venue** evidence:

`Provider → Normalizer → Evidence → Workspace → Report`

without changing Rule / Projection / Feature math.

### Delivered path

| Stage | Change |
|---|---|
| Provider | `FootballVenue` on `FootballFixture`; map `fixture.venue` from API-Football; recorded cassettes include venue; `toEvidenceMatchShape` emits `venue` |
| Normalizer | Optional `parseVenue` → Evidence type **`VENUE`** with provider metadata |
| Evidence | New type `VENUE`; registry capability `venue.ingestImplemented=true` for `football:api-sports` |
| Workspace | Venue section + hero venue label + Evidence timeline VENUE icon/title |
| Report | Explainable report surfaces venue from Evidence (presentation only; not Projection) |

### Explicitly not changed

- `@fas/feature` Feature extraction
- `@fas/rule` Rule evaluation
- `@fas/analysis` DeterministicMatchProjection
- Injury / Lineup / Player ingest

### Honest absence

If `venue` is missing from provider shape, **no** `VENUE` Evidence is emitted (not “unknown stadium” invented).
