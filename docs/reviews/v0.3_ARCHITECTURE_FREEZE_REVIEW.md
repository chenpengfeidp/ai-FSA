## 5. Confidence audit

### Verdict: **PASS**

**Policy:** `confidence.mvp.a05` (unchanged — no Confidence redesign in I2B).

Confidence is composed from:

1. **Evidence completeness** — structured Presence checks over sealed Evidence types/splits  
2. **Rule agreement** — P1 football channel weight dominance + contradiction penalty  
3. **Intelligence agreement bonus** — xG / Advanced / Context cross-support (capped)  
4. **Market alignment** — supporting bonus when Market lean agrees with Football; penalty when they disagree (capped); Market does **not** override Football  
5. **Scenario concentration / upset** — from sealed ScenarioSet  

**No prediction shortcuts:** Confidence does not recompute 1X2 from raw Features or Provider data. Feature bundle appears only as status/checksum limitation metadata.

## 6. Projection audit

### Verdict: **PASS** with **accepted dual-input boundary** (**WARN** if misread as “Rules-only λ”)

### 6.1 How 1X2 is formed

1. **Poisson baseline λ** from Feature ratings (`attackRating*`, `defenseRating*`, momentum, homeAdvantage)  
2. **Football channel Rules** (`home+` / `away+` PASS weights) → delta → `softmaxAdjust`  
3. **Calibration artifact** reference (no in-run training)  
4. Market lean conflict → recommendation **cautious** (gate), not probability blend  
5. I2B Market Intelligence PASS → **limitations text only**

### 6.2 Market never directly sets football probabilities

Confirmed:

- Market Rules excluded from `footballChannelRules`  
- Explicit limitation: market signals do not blend into 1X2  
- Steam / Consensus / RLM / Volatility / Sharp do not adjust softmax  

### 6.3 Accepted Freeze boundary (ratified)

Strict reading “Projection consumes only Rule outputs” is **not** how FAS v0.2 was designed. The accepted contract is:

> **Features** supply the independent Poisson baseline and strength component **S**.  
> **Football Rules** supply directional 1X2 adjustment.  
> **Market Rules** supply findings / conflict gates / limitations only.

This review **ratifies** that dual-input model for v0.3. Changing it would be an architecture redesign (out of scope).
