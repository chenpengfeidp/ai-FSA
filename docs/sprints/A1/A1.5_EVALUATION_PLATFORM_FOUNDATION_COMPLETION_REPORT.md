## 1. Completion Report

A1.5 adds a reusable **Evaluation History** platform layer on top of A1 per-match evaluation:

```text
Prediction → Actual → Evaluation → Evaluation History (append-only)
```

Historical records never rewrite sealed Projection / Prediction. Future A2 Calibration (and later Statistics / Knowledge / Case) must consume History, not live Prediction seals.

## 3. Evaluation History model

Preserved fields:

| Field | Source |
|---|---|
| Match ID | sealed prediction |
| Competition | Actual `MATCH_RESULT` |
| Season | matchDate UTC year (honest default) |
| Match Date | MATCH_INFO kickoff |
| Home / Away | MATCH_INFO / Features |
| Prediction Snapshot | sealed `SealedPredictionInput` copy |
| Actual Result | `ActualMatchResult` |
| Evaluation Metrics | scored `PredictionEvaluationRecord` |
| Confidence | predictionConfidence + band snapshot |
| Feature / Rule / Projection / Evaluation versions | pin lineage |
| Timestamp | `recordedAt` |

`historyId` is deterministic (`matchId + projectionChecksum + evaluation.checksum`) for idempotent append.

---

## 4. Storage implementation

| Mode | Backend |
|---|---|
| Default (`EVIDENCE_REPOSITORY_MODE=memory`) | process-local `InMemoryEvaluationHistoryRepository` |
| `EVIDENCE_REPOSITORY_MODE=postgres` | Prisma `evaluation_history_items` |

Save is append-only; identical seal re-save returns the existing row.
