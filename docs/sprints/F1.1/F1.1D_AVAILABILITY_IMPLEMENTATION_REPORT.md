## Implementation Report

### Goal

End-to-end **Availability** facts:

```text
Provider → Normalizer → Evidence → Workspace → Report
```

Slices:

1. **Injury** Evidence  
2. **Suspension** Evidence  
3. **Availability Summary** (composed Injury + Suspension)

Without Feature / Rule / Projection / Analysis changes. Without Expected Lineup.

### Delivered path

| Stage | Change |
|---|---|
| Provider | `FootballAvailabilityAbsence`; map `/injuries`; live fetch by fixture; cassette rows on `football:100001` |
| Normalizer | `parseAvailabilityAbsences` → `INJURY` / `SUSPENSION` |
| Evidence | Additive type `SUSPENSION`; registry `injury.ingestImplemented=true` |
| Workspace / Report | Availability section + summary counts; timeline titles/icons |
| Catalog | `docs/50_EVIDENCE_CATALOG.md` status for INJURY / SUSPENSION / AVAILABILITY |

### Honest absence

- Empty or missing provider data → **no** Injury/Suspension Evidence.
- UI states Availability as **not available**.
- Does **not** invent Unknown, “0 absences”, or “everyone available”.
- Unknown provider `type` strings are skipped (not guessed).

### Explicitly not changed

- `@fas/feature`, `@fas/rule`, `@fas/analysis`
- Expected Lineup, confirmed Lineup, Referee, richer Form
- Engines / Architecture Freeze edges

| Analyze minimum | Still MATCH_INFO + TEAM_FORM×2 + STATISTICS×2 |
| Existing Evidence types | Unchanged semantics |
| Postgres envelope | New type stored in `value_json`; no schema migration |

## Migration Notes

1. No Prisma / SQL migration.
2. Re-import recorded `football:100001` (or re-run normalize) to obtain Injury/Suspension Evidence.
3. Clients must accept Evidence type `SUSPENSION`.
4. Live path adds `/injuries?fixture=` (quota cost); empty/fail → honest absence.
